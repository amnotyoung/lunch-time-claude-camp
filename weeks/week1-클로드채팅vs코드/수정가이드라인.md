## 수정 사항

### 1. Slide 02 — 한계점 1: "파일 접근" 표현 보정

**문제**: 클로드 채팅도 파일 업로드(드래그&드롭, 첨부)를 통해 파일 내용을 읽을 수 있다. 현재 문구는 파일을 아예 볼 수 없는 것처럼 읽힐 수 있다.

**현재 문구**
```
제목: 내 파일에 직접 접근 불가
설명: 코드를 복사 → 붙여넣기 → 다시 복사... 반복
```

**수정 문구**
```
제목: 내 파일을 직접 수정할 수 없다
설명: 파일을 올려서 읽을 순 있지만, 결과를 직접 저장해주진 않는다
```

**수정 근거**: 채팅은 "읽기"는 가능하지만 "쓰기(수정·저장)"가 불가능한 것이 정확한 한계. 발표 중 청중이 "저는 파일 올려서 쓰는데요?"라고 질문할 수 있으므로 선제적으로 보정.

### 2. Slide 02 — 한계점 2: "맥락 사라짐" 표현 보정

**문제**: 클로드 채팅에도 프로젝트(Projects) 기능과 메모리가 도입됨. "대화가 끝나면 맥락이 사라진다"는 점점 부정확해지고 있다.

**현재 문구**
```
제목: 대화가 끝나면 맥락이 사라짐
설명: 매번 처음부터 다시 설명해야 함
```

**수정 문구**
```
제목: 프로젝트 단위의 깊은 맥락 유지가 어렵다
설명: 프로젝트 구조, 코딩 규칙 등 세밀한 맥락까지는 기억 못함
```

**수정 근거**: 채팅의 메모리 기능과 차별화하면서도, 클로드 코드의 CLAUDE.md 기반 맥락 유지가 더 강력하다는 점을 정확히 전달.

### 3. Slide 05 — pytest 명령어 플래그 수정

**문제**: `--coverage`는 pytest의 유효한 플래그가 아니다. pytest-cov 플러그인에서 사용하는 올바른 플래그는 `--cov`.

**현재 문구**
```
$ pytest tests/ --coverage     테스트 실행
```

**수정 문구**
```
$ pytest tests/ --cov          테스트 실행
```

**수정 근거**: 비개발자 청중이라 당장 문제는 안 되지만, 개발자가 섞여 있을 경우 신뢰도에 영향. 올바른 플래그를 쓰는 것이 비용 없이 정확도를 높임.

### 4. Slide 06 — Desktop Commander 기능 설명 보정

**문제**: "앱 자동화 · 스크린샷"은 Desktop Commander의 핵심 기능이 아니다. 이 도구는 주로 파일 시스템 접근, 프로세스 관리, 터미널 명령 실행에 특화되어 있다.

**현재 문구**
```
기능 목록:
- 파일 탐색 및 읽기·쓰기
- 실행 중인 프로세스 관리
- 앱 자동화 · 스크린샷
```

**수정 문구**
```
기능 목록:
- 파일 탐색 및 읽기·쓰기
- 실행 중인 프로세스 관리
- 터미널 명령 실행·자동화
```

**수정 근거**: 실제로 설치 후 사용해볼 청중이 있을 수 있으므로, 기대치를 정확하게 맞추는 것이 중요. 스크린샷 기능은 별도 MCP (예: Claude in Chrome)의 영역.

---

### 5. Slide 07 — Memory 설명 정밀도 개선

**문제**: 현재 문구는 전체 대화 내용이 세션 간에 이어지는 것처럼 읽힌다. 실제로는 Memory가 특정 핵심 정보(사용자 선호, 프로젝트 관련 사실 등)를 `~/.claude/memory.json`에 저장하는 방식이며, 전체 대화 이력이 복원되는 것은 아니다.

**현재 문구**
```
제목: 대화 맥락 유지
설명: 세션이 끊겨도 기억을 이어서 사용
부연: 다음 날 열어도 OK
```

**수정 문구**
```
제목: 핵심 정보 기억
설명: 중요한 사실·선호를 세션 간 기억
부연: "지난번 얘기한 DB 구조" 기억함
```

**수정 근거**: "대화 맥락 유지"는 전체 대화가 이어지는 것으로 오해될 수 있다. 실제로는 핵심 팩트 저장 방식이므로, 기대치를 정확히 맞추고 나중에 "왜 대화가 안 이어지지?"라는 혼란을 방지.
