- **Topic**: 클로드 채팅과 클로드 코드의 차이점 — 클로드 코드의 4가지 핵심 차별점 중심

## Slide Composition

### Slide 1 - Cover
- **Type**: Cover
- **Title**: 클로드 채팅과 클로드 코드는 뭐가 다른가
- **Subtitle**: 점심시간 클로드 캠프 #1

### Slide 2 - 우리가 아는 클로드 채팅
- **Type**: Content
- **Key Message**: 클로드 채팅은 강력하지만, 한계가 있다
- **Details**:
  - 한 줄 요약: "브라우저에서 대화하는 AI"
  - 한계점 3가지:
    - 내 파일에 직접 접근 불가 → 복사·붙여넣기 필수
    - 대화가 끝나면 맥락이 사라짐
    - 결과물을 내가 직접 적용해야 함

### Slide 3 - 클로드 코드는 다르다
- **Type**: Section Divider (임팩트 강조)
- **Key Message**: "AI가 내 컴퓨터에서 직접 일한다"
- **Details**:
  - 채팅과의 가장 근본적인 차이를 한 문장으로 전달
  - 이후 4가지 차이점의 도입부 역할

### Slide 4 - 차이점 1: 파일을 직접 읽고 쓴다
- **Type**: Content (Split Layout)
- **Key Message**: 복붙 없이 내 파일을 바로 편집
- **Details**:
  - Before: 채팅에서 코드 복사 → 에디터에 붙여넣기
  - After: 클로드 코드가 파일을 직접 열고, 읽고, 수정
  - 로컬 파일시스템 접근 가능

### Slide 5 - 차이점 2: 명령어를 직접 실행한다
- **Type**: Content
- **Key Message**: AI가 터미널을 대신 쓴다
- **Details**:
  - 터미널 명령어 실행 (npm install, git commit 등)
  - 빌드·테스트·배포까지 직접 수행
  - 사람이 할 일을 AI가 대신

### Slide 6 - 차이점 3: 도구를 연결할 수 있다 (MCP)
- **Type**: Content
- **Key Message**: MCP로 외부 도구를 AI에 연결
- **Details**:
  - MCP(Model Context Protocol) 한 줄 설명
  - 예시 1 — koica-reg: KOICA 인사·회계 규정 조회·키워드 검색
  - 예시 2 — SQLite: 데이터베이스에 직접 질의하고 결과 분석
  - 연결할수록 클로드 코드가 할 수 있는 일이 늘어남

### Slide 7 - 차이점 4: 기억하고 학습한다
- **Type**: Content
- **Key Message**: 프로젝트 맥락을 기억해 매번 설명 안 해도 된다
- **Details**:
  - CLAUDE.md: 프로젝트 규칙·스타일 저장
  - 메모리: 대화 간 맥락 유지
  - 스킬: 반복 작업을 명령어 하나로

### Slide 8 - 실제 활용 예시
- **Type**: Statistics / Content
- **Key Message**: 채팅으로는 어렵고, 코드로는 쉬운 것들
- **Details**:
  - 예시 A: 100개 파일 일괄 변환 → 클로드 코드가 직접 처리
  - 예시 B: DB 데이터 분석 요청 → SQLite MCP로 즉시 질의
  - 예시 C: 매일 반복하는 보고서 작성 → 스킬로 자동화
