## 사실 근거 (공식 문서 code.claude.com/docs)
- 팀원 = 독립 컨텍스트 윈도우 → 팀장 대화·옆 팀원 머릿속 못 봄 (오염 방지의 원리)
- **분담(기본·설정 불필요)**: fan-out subagent — 결과만 팀장에게 보고 ↔ **소통형 팀(실험·설정 필요)**: 팀원끼리 직접 메시지 + 공유 할 일 목록
- **agent teams는 실험 기능, 기본 비활성** → settings.json에 `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` (Claude Code v2.1.32+) 필요. SendMessage/TeamCreate도 이때 활성
- 팀원은 CLAUDE.md·MCP·스킬 자동 로드, 단 팀장의 대화 이력은 상속 안 함 → 공통 기준은 CLAUDE.md에
- (speaker note) "CLAUDE.md 자동 로드"는 팀 teammate 기준. 일부 built-in subagent(Explore/Plan)는 CLAUDE.md·부모 git status 건너뜀
- 팀 vs 혼자(문서): 분담=빠른 결과만, 소통형 팀=서로 findings 공유·challenge·coordinate, 순차/같은 파일/의존 많으면 단일 세션·분담이 유리

## 실습 검증 완료 (소통형 팀, 실제 실행)
플래그 설정 후 'koica-review' 팀 생성 → 예산·규정·성과 팀원 3명 실행. **팀원 간 peer DM 3방향 실제 발생**(규정→예산 인건비, 성과→규정 TTMS, 예산→성과 지속가능성). 대화가 도메인을 연결해 **분담만으론 안 나올 교차 발견** 도출: "사후 운영비 공백 → 성과 3년 내 소실(성과)·수원국 자체재원 약정 미충족(규정)·유지보수 재원 부재(예산)". koica-reg 실재 조문 인용(조달 및 계약규정 제5·15조, 전문인력 업무처리 지침 제7·17조). ⚠ 일부 세부수치·조문번호는 팀장이 검증(koica-reg verify/get).

### Slide 02 — 한 명이 아니라 팀, 왜?

- 푸터: 매주 써온 Claude도 이미 에이전트 — 이제 그 에이전트가 팀장이 되어 팀원을 부린다

### Slide 03 — 머릿속은 따로, 연락은 가능 (핵심)
- 머릿속은 따로(언제나): 독립 컨텍스트 → 오염 방지 + 병렬
- 연락은 가능(오늘 실습·설정 필요): 메시지·공유 할 일 목록으로 의논·교차 검토
- 구분: 분담(기본·설정 불필요, 결과만) ↔ 소통형 팀(오늘, `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1`)

### Slide 04 — 두 가지 운용: 분담 vs 소통형 팀 (구조 비교)
- 공식 Subagents↔Agent Teams 비교도를 데크 스타일(한글)로 재구성. 위상(topology) 대비가 핵심
- 분담(서브에이전트): 팀장→일꾼들(띄움)→결과만 보고(별 모양). 일꾼끼리 연결 없음. 기본·설정 불필요
- 소통형 팀(에이전트팀): 팀장→공유 할 일 목록, 팀원끼리 ↔ 메시지로 자체 조율(그물망). 실험·설정 필요
- 한 줄: 빠른 취합=분담 / 서로 의논해 엮기=소통형 팀
- slide 03(분담↔소통형 팀)의 시각적 심화. 다음 CLAUDE.md·R&R·실습의 토대

### Slide 05 — CLAUDE.md = 팀 공통 운용 매뉴얼

- 적는 것: 호칭·용어·금지·산출물 형식·경로/카테고리표

### Slide 06 — 역할 분배(R&R)는 팀장이 알아서

- 원하면 "예산·규정·성과로 나눠"처럼 짚어 통제 가능(안 짚어도 됨)
- 좋은 R&R: 독립적·경계 분명 / 곁다리: 나만의 에이전트로 저장

### Slide 07 — 실습: 서로 의논하는 검토팀
- 발화: "에이전트팀으로 검토 — 예산·규정·성과로 나누고, 겹치는 이슈는 팀원끼리 의논해서(koica-reg·CLAUDE.md 참고) 하나의 보고서로"
- 비주얼: 팀장 → 예산⇄규정⇄성과(팀원끼리 메시지) → 종합. "팀원끼리 의논"이 소통형 팀 트리거
- 사전 준비: settings.json `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` (v2.1.32+)
- 교차 통찰(실증): 사후 운영비 공백 → 성과 소실·자체재원 약정 미충족·유지보수 재원 부재 (분담만으론 안 이어질 연결)

### Slide 08 — 팀원끼리 오간 실제 DM (실습 결과)
