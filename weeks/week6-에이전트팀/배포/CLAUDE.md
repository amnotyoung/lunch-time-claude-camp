# 점심시간 클로드 캠프 — 팀 작업 규칙 (CLAUDE.md)

이 문서는 **본체(팀장)와 서브에이전트(팀원)가 함께 따르는 공통 기준**이다.
새로 합류한 팀원은 이 문서만 읽으면 같은 기준으로 일할 수 있다.
※ CLAUDE.md는 **잠금장치가 아니라 기준 문서** — 강제가 아닌 공통 약속이다.

## 작성 규칙
- 한국어로 쓴다. 기술 용어·식별자는 원문 그대로 둔다.
- 근거 없이 **개인 이름·소속 등 개인정보(PII)를 지어내지 않는다.** 모르면 추측하지 말고 묻는다.

## 용어
- **koica-reg** — KOICA 규정 검색 도구. 로컬 저장소 `~/Documents/Claude/Projects/koica-reg-mcp` (`python3 koica_search.py search "..."` / `get` / `verify`).
- **ktm** — 규정 텍스트마이닝 스킬(조회→키워드→그래프→보고서).
- **분담** — 서브에이전트 fan-out, 결과만 보고(설정 불필요). **소통형 팀** — 팀원끼리 메시지·공유 할 일 목록(`CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1`, Claude Code v2.1.32+, 재시작 필요).

## 산출물 형식
- 보고서·정리물은 **마크다운(.md)**.
- 주장에는 **근거(문서 페이지 / 규정 조문)** 를 함께 표기한다.
- 표가 도움이 되면 표로, 핵심은 한 줄 결론으로.

## 금지·주의사항
- **규정 조문·조문번호·수치를 단정하지 않는다.** 반드시 koica-reg `get`/`verify`로 원문을 확인한 뒤 인용한다(번호·개정일 포함).
- 출력물(`outputs/`·`*.png`·`*.pdf`)과 캐시는 버전 관리 대상이 아니다.
- 외부로 내보내는 작업(공유·전송)은 동료 확인 후 진행한다.

## 검토팀(에이전트팀) 운용 기준
- **역할(R&R)은 경계를 분명히** 나눈다. 기본 3역할: **예산 / 규정 / 성과**. 겹치면 중복, 의존하면 순차가 되므로 따로 떼어도 되는 단위로 나눈다.
- **규정 담당**은 koica-reg 조문과 대조하고 **출처(규정명·조문번호)** 를 반드시 명시한다.
- 도메인이 겹치는 **교차 이슈는 팀원끼리 의논**(소통형 팀)해 연결한다.
- 결과는 **하나의 종합 보고서**로 합친다.
- 팀원이 인용한 규정·수치는 **팀장이 koica-reg로 검증**한 뒤 확정한다.
- 팀은 공짜가 아니다 — 독립·다관점·양 많은 일에만 팀, 짧은·순차·같은 파일 작업은 혼자.
