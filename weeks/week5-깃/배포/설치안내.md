# 5주차 실습 준비 — 설치 안내

이번 주(깃) 실습 대상은 **week4에서 만든 `ktm` 스킬**입니다. 이 스킬을 git으로 버전 관리합니다.
아래 두 가지를 미리 설치해두면 실습이 매끄럽습니다. (대부분 **Claude Code에게 한국어로 부탁**하면 알아서 처리합니다.)

---

## 들어 있는 것

| 파일 | 무엇 |
|---|---|
| `ktm.zip` | 실습 대상 스킬 폴더 (조회·키워드·그래프·보고서 4단계) |
| `koica-reg-mcp.zip` | ktm이 참조하는 KOICA 규정 데이터 + MCP 서버 (인덱스 **사전 빌드 포함** → 빌드 불필요) |

> 3주차에 koica-reg를 이미 연결한 분은 **A단계를 건너뛰어도** 됩니다. 빠졌거나 새로 까는 분만 A부터.

---

## A. koica-reg-mcp 설치 + 연결

1. **압축 풀어 폴더 두기** — `koica-reg-mcp.zip`을 풀어 아래 위치에 둡니다(경로 자동 인식 위치):
   ```
   ~/Documents/koica-reg-mcp/
   ```
   (또는 `~/Documents/Claude/Projects/koica-reg-mcp/` 도 자동 인식됩니다.)

2. **MCP로 연결** — Claude Code에 그대로 입력:
   > `~/Documents/koica-reg-mcp/koica_mcp_server.py` 를 **koica-reg 라는 이름의 MCP로 연결해줘.** 필요한 `mcp` 패키지도 설치해줘.

   참고로 뒤에서 도는 명령은 이렇습니다(직접 칠 필요 없음):
   ```bash
   pip install "mcp>=1.0"
   claude mcp add koica-reg -- python3 ~/Documents/koica-reg-mcp/koica_mcp_server.py
   ```

3. **확인** — Claude Code에서 `/mcp` 입력 → 목록에 `koica-reg` 가 보이면 성공.

> 인덱스(`data/index.json`, 조문 1,660개)가 이미 들어 있어 **빌드 단계는 필요 없습니다.**

1. **압축 풀어 폴더 두기** — `ktm.zip`을 풀어 아래 위치에 둡니다:

   (잘 풀렸다면 그 안에 `SKILL.md`, `scripts/`, `examples/` 가 보입니다.)
