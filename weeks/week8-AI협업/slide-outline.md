# 클로드와 챗지피티를 협업시켜보자 — 두 AI가 함께 만든 8주의 마무리

## Meta
- **Topic**: 캠프 피날레. 지난 7주를 **주차당 1장**으로 돌아보는 wrap-up을 **먼저** 보여주고(복습), 그 wrap-up 슬라이드가 실은 **클로드(제작)와 챗지피티=코덱스(검수)의 협업 산출물**임을 밝힌 뒤 그 과정을 retrospect로 회고한다. GitHub를 무대로 두 회사의 AI가 협업.
- **핵심 대비(데크 축)**: week6 = 같은 Claude끼리 팀(**동종**) ↔ 오늘 = 다른 회사 AI 둘(클로드+챗지피티, **이종**). 같은 AI는 같은 맹점 / 다른 AI는 다른 눈 → **교차검증**. 생산=클로드 · 검수=코덱스 · 판단=사람.
- **협업 메커니즘**: 내가 **push/PR** → `@codex review` → **코덱스가 클라우드에서 PR을 자동 리뷰**(로컬 호출·worktree 없음). 사람은 트리거만, 두 AI는 GitHub(이슈/PR/리뷰)로 소통.
- **자기참조(중요)**: 이 강의 슬라이드 자체가 협업의 결과물. 코덱스 검수 = "클로드의 7주 요약이 **원본 각 주차 슬라이드와 맞는가**" 대조 → 이종 AI 교차검증의 이상적 사례.
- **CLAUDE.md ↔ AGENTS.md 대칭(핵심 통찰)**: week6 CLAUDE.md(클로드 팀 공통 기준)와 똑같이, **AGENTS.md = 코덱스에게 주는 리뷰 기준**. "AI에게 파일로 기준을 준다"는 같은 원리.
- **Target Audience**: week1~7을 누적 수강한 비개발자 KOICA 동료 (채팅vs코드 · 환경 · MCP · 스킬 · git · 에이전트팀 · GitHub까지 직접 해봄)

## 사실 근거 (실제 확인)
- **챗지피티 = Codex**(OpenAI). `codex` 설치(0.125.0)·"Logged in using ChatGPT". (week3 곁다리에서 한 번 소개)
- **Codex GitHub code review**(OpenAI 정식 기능) = 내가 push/PR하면 **코덱스가 클라우드에서 PR diff를 자동 리뷰**. PR 코멘트 `@codex review`로 트리거, `@codex fix it`로 수정까지. 이 환경에 GitHub 통합 플러그인 `github@openai-curated` **활성화**됨. (로컬 CLI 호출이 아니라 **push가 트리거**)
- **AGENTS.md** = 코덱스에게 주는 **리뷰 기준 파일**(repo 최상위). week6 **CLAUDE.md(클로드 팀 기준)와 대칭** — 'AI에게 파일로 기준을 준다'는 같은 원리. 여기에 "wrap-up이 원본 주차 슬라이드와 사실이 맞는지 검수" 지침을 적는다.
- **두 AI는 직접 대화하지 않음** — GitHub PR/리뷰 코멘트가 회사 다른 두 AI의 **공통 언어**(week7에서 배운 그것).
- 실습 repo = 이 캠프 자료 repo 자체 `github.com/amnotyoung/lunch-time-claude-camp`. week1~7 슬라이드(HTML) 전부 완성 → 코덱스가 대조할 원본이 실재.
- **사람이 할 일(웹 설정)**: Codex가 이 repo에 연결돼 있어야 함 — chatgpt.com/codex에서 Code review 연결(또는 PR에 `@codex review` 멘션). 클로드(코드 탭)는 코덱스의 클라우드 리뷰를 대신 못 켬.

## 역할별 실행 주체
- **클로드(코드 탭, 자동)**: wrap-up 슬라이드 제작 · 단계별 커밋 · push · PR 생성 · 코덱스 리뷰 반영 · 머지.
- **코덱스(클라우드, GitHub 통합)**: PR을 감지해 원본↔요약 대조 검수 → PR에 리뷰 코멘트. 기준은 **AGENTS.md**.
- **사람**: push 후 PR에 `@codex review` 한 줄 → 두 AI가 GitHub로 협업. 최종 머지 판단.
