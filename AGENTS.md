# AGENTS.md — 코덱스(Codex) 리뷰 기준

> 이 파일은 **Codex의 GitHub code review**가 따르는 기준이다. week6의 `CLAUDE.md`(클로드 팀 공통 기준)와 대칭 — "AI에게 파일로 기준을 준다"는 같은 원리. 점심시간 클로드 코드 캠프 8주차(클로드×챗지피티 협업) 실습에서 코덱스가 클로드의 PR을 검수할 때 이 기준을 적용한다.

## 이 저장소가 무엇인가
- **점심시간 클로드 코드 캠프** 강의자료. 비개발자 KOICA 동료가 자연어로 클로드 코드를 익히는 8주 과정.
- 슬라이드는 주차별 폴더 `weeks/weekN-주제/`에 **독립 HTML**(`slide-01.html` … )로 보관. 설계는 `slide-outline.md`.
- 슬라이드 규격: **720pt × 405pt 고정(16:9)**, Bento Grid, 디자인 토큰 — navy #1A1A2E / lime #E8FF3B / teal #4ECDC4 / yellow #FFE66D / pink #FF6B9D / off-white #F8F8F2 / gray #A0A0B0.

## 8주차 wrap-up 검수 — 가장 중요
8주차 슬라이드(`weeks/week8-AI협업/`)는 **지난 7주(week1~7)를 주차당 1장으로 요약**한 것이다. 리뷰의 핵심은 **요약이 원본과 사실이 맞는지** 대조하는 것:

- **P0 (반드시 지적)**: wrap-up 슬라이드의 서술이 **원본 주차 슬라이드(`weeks/week1~7/slide-*.html`)·outline의 사실과 어긋남** — 틀린 사실, 왜곡, 숫자 오류, 핵심 개념 오인. (예: week3 요약에서 koica-reg를 다른 도구로 적음, week5 핵심을 잘못 요약)
- **P0**: 오타·맞춤법·띄어쓰기.
- **P1**: 원본의 **핵심 메시지 누락** — 그 주차를 한 장으로 대표하기에 빠지면 안 되는 것이 빠짐.
- **P1**: 과장·오해 소지 표현(특히 "두 AI가 직접 대화한다"는 식의 메커니즘 오해).
- **P2**: 디자인 토큰·규격(720×405pt, 색상) 이탈, 슬라이드 간 톤 불일치.

## 대조 기준
- 각 wrap-up 슬라이드는 대응 원본과 맞춰 본다: slide-03↔week1, 04↔week2, 05↔week3, 06↔week4, 07↔week5, 08↔week6, 09↔week7.
- 원본 근거는 각 주차의 `slide-outline.md`와 `slide-*.html`. 근거가 모호하면 단정 말고 "확인 필요"로 표시.

## 리뷰 작성 방식
- **한국어**로, 슬라이드 파일·위치를 짚어 구체적으로.
- 심각도(P0/P1/P2)를 붙이고, 무엇이 왜 틀렸는지 + 원본의 올바른 내용을 제시.
- 추측성 지적 자제 — 원본 대조로 확인되는 것 위주.

## 톤·용어 규율 (캠프 공통)
- "학생" 금지 → "동료" 또는 주어 생략.
- 비개발자 대상 — 과한 전문용어 지적보다 **사실 정확성·이해 가능성** 우선.
